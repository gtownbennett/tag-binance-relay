TAG TERMINAL PREDICTION LEDGER v2.5.0

Forecast horizons
- 6 hours
- 24 hours
- 3 days
- 7 days

Automatic behavior
- Every successful Chad analysis records four machine-readable horizons.
- A background task checks matured forecasts every LEDGER_AUTO_GRADE_SECONDS.
- Grading uses the Binance futures 5-minute close nearest the due time.
- Direction, target-range hit, invalidation, midpoint error, score and post-mortem are saved.
- After every save or grade, an atomic JSON backup is written to LEDGER_BACKUP_PATH.
- If SQLite is empty at startup and the backup exists, the service restores it automatically.

Learning safeguards
- Calibration is inactive until at least eight horizons are graded.
- Probability bins compare stated probability with realized direction accuracy.
- Weak horizons receive lower confidence multipliers.
- A deterministic confidence cap is applied only after enough outcomes exist.
- Stale or missing live data can impose a stricter confidence cap.

Recovery
- GET /v1/chad/ledger/export creates a portable JSON backup.
- POST /v1/chad/ledger/import merges or replaces from that export.
- This allows TAG Terminal's on-device copy to restore the server after a redeploy.
- Full server-side durability still requires a persistent disk because /tmp can be erased.

Transparency
- GET /v1/chad/changes explains why the current durable view changed.
- GET /v1/chad/calibration exposes confidence calibration.
- GET /v1/tag/freshness exposes source age and stale-data warnings.
